**2.1 Техническое задание**

Техническое задание является основным документом, определяющим цели, назначение и требования к разрабатываемому программному продукту. Техническое задание разработано в соответствии с требованиями ГОСТ 19.201–78 и устанавливает состав функций автоматизированной системы обмена криптовалют, условия её эксплуатации и порядок приёмки.

Разрабатываемая автоматизированная система предназначена для комплексной автоматизации деятельности сервиса обмена криптовалют. Система должна обеспечивать хранение информации о пользователях, курсах криптовалют, заказах, платёжных реквизитах и транзакциях. Использование программного продукта позволит сократить время обработки заказов, повысить эффективность работы сотрудников и улучшить качество обслуживания клиентов.

Основной целью разработки является создание единой информационной среды, обеспечивающей автоматизацию основных бизнес-процессов сервиса обмена криптовалют. Для достижения поставленной цели необходимо реализовать механизмы регистрации и авторизации пользователей, управление заказами на обмен, интеграцию с криптокошельками, проверку блокчейн-транзакций, формирование отчётности и обмен сообщениями между клиентами и службой поддержки.

Функциональные возможности системы должны обеспечивать регистрацию новых пользователей, разграничение прав доступа в зависимости от роли пользователя, ведение справочника курсов криптовалют, создание заказов с автоматическим расчётом сумм, оплату через криптокошелёк Phantom, проверку статуса транзакций в блокчейне Solana, хранение информации о платежах и формирование отчётной информации. Администратор системы должен иметь возможность управления всеми объектами системы, в то время как обычные пользователи получают доступ только к функциям, предусмотренным их ролью.

К разрабатываемому программному продукту предъявляются требования по надёжности, производительности и безопасности. Система должна обеспечивать стабильную работу при одновременном использовании несколькими пользователями, сохранять целостность данных и предотвращать несанкционированный доступ к информации. Для защиты учетных записей пользователей предусматривается использование механизмов аутентификации и безопасного хранения токенов авторизации.

В качестве программной платформы для реализации системы используется связка FastAPI (Python) для серверной части и JavaScript для клиентской. Для хранения информации применяется система управления базами данных SQLite. Пользовательское взаимодействие осуществляется через веб-интерфейс, разработанный с использованием HTML, CSS и JavaScript. Эксплуатация системы предполагается на персональных компьютерах под управлением операционной системы Windows с установленным современным веб-браузером и доступом к сети Интернет.

Разработка программного продукта включает этапы анализа предметной области, проектирования структуры базы данных, разработки пользовательского интерфейса, реализации серверной части, тестирования и внедрения системы. Контроль выполнения работ осуществляется путём проверки работоспособности реализованного функционала и соответствия полученного результата требованиям технического задания.

Приёмка программного продукта производится после завершения тестирования. Основным критерием приёмки является корректное выполнение системой всех предусмотренных функций, отсутствие критических ошибок и соответствие реализованного программного обеспечения установленным требованиям.

**2.2 Алгоритм решения задачи**

Алгоритм решения задачи определяет последовательность обработки данных в автоматизированной системе обмена криптовалют и порядок взаимодействия пользователей с программным продуктом. Разрабатываемая система реализована в виде веб-приложения на базе FastAPI и функционирует по клиент-серверной архитектуре. В качестве средства хранения данных используется система управления базами данных SQLite.

Работа системы начинается с обращения пользователя к веб-приложению через браузер. После открытия главной страницы пользователь получает доступ к общедоступной информации о курсах криптовалют и форме создания заказа. Для использования полного функционала системы, включая оплату через криптокошелёк и общение с поддержкой, необходима авторизация администратора.

В процессе создания заказа пользователь выбирает актив (USDT или SOL), вводит сумму, выбирает валюту и банк для получения средств. Система автоматически запрашивает актуальный курс через внешний API и рассчитывает сумму к получению с учётом комиссии сервиса. После проверки корректности заполнения всех полей создаётся новый заказ, генерируется уникальный адрес для оплаты, и информация сохраняется в базе данных.

После создания заказа система отображает пользователю платёжные реквизиты — криптовалютный адрес и сумму к оплате. Для актива SOL дополнительно отображается QR-код для удобства сканирования. Пользователь может оплатить заказ, отправив криптовалюту на указанный адрес через любой криптокошелёк. При наличии установленного расширения Phantom пользователь может оплатить напрямую через интерфейс системы, используя интеграцию с плагином.

При оплате через Phantom система выполняет следующие действия: подключается к кошельку пользователя, создаёт транзакцию перевода на депозитный адрес сервиса, запрашивает подписание транзакции пользователем и отправляет подписанную транзакцию в блокчейн-сеть. После отправки система ожидает подтверждения транзакции и обновляет статус заказа.

Для проверки поступления средств система периодически выполняет запросы к блокчейн-провайдеру с целью получения информации о транзакциях на депозитный адрес. При обнаружении новой транзакции система сверяет сумму перевода с ожидаемой и, в случае совпадения, обновляет статус заказа на «оплачен». При несовпадении суммы или отсутствии транзакции в течение установленного времени заказ автоматически отменяется.

Для обеспечения взаимодействия между клиентами и службой поддержки в системе реализован чат. После начала чата создаётся новая сессия, и пользователь получает возможность отправлять сообщения через WebSocket-соединение. Все сообщения сохраняются в базе данных и отображаются в интерфейсе в режиме реального времени. Администратор может просматривать активные чат-сессии и отвечать клиентам.

Администратор системы имеет доступ к панели управления, где может просматривать список всех заказов, фильтровать их по статусу, просматривать детальную информацию по каждому заказу и изменять статусы обработки. Для доступа к административной панели требуется авторизация с использованием секретного токена.

После завершения выполнения необходимых операций пользователь может завершить работу с системой путём закрытия веб-страницы. При необходимости администратор может выйти из учётной записи через интерфейс панели управления.

Таким образом, разработанный алгоритм обеспечивает выполнение всех основных бизнес-процессов сервиса обмена криптовалют, включая управление заказами, интеграцию с криптокошельками, проверку блокчейн-транзакций, поддержку клиентов и формирование отчётности.
