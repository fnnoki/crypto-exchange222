**1.1 Обзор предметной области**

В настоящее время сфера цифровых финансов продолжает активно развиваться, а криптовалюты становятся всё более популярным инструментом для проведения транзакций и сохранения капитала. Современные сервисы по обмену цифровых активов предоставляют широкий спектр услуг, включающий обмен криптовалют, вывод средств на банковские карты, пополнение криптокошельков и сопровождение клиентов на всех этапах сделки. Увеличение количества пользователей и объёмов транзакций приводит к необходимости автоматизации внутренних процессов организации.

Основной задачей сервиса обмена криптовалют является организация и сопровождение сделок между продавцами и покупателями цифровых активов. Для проведения обмена необходимо учитывать большое количество факторов: актуальный курс обмена, доступность платёжных систем, лимиты на операции, контроль статуса заказа и взаимодействие между участниками сделки. При ручном управлении подобными процессами существенно возрастает вероятность ошибок, связанных с неверным расчётом сумм, потерей информации о заказах и отсутствием актуальных данных о курсах валют.

Дополнительной сложностью является учёт платёжных реквизитов и обеспечение безопасности транзакций. Каждая криптовалютная сделка требует генерации уникального адреса для оплаты, проверки поступления средств и подтверждения транзакции в блокчейн-сети. Отсутствие централизованной системы учёта затрудняет контроль финансовых потоков, отслеживание статусов оплат и ведение истории операций.

В рамках деятельности сервиса важную роль играет работа с клиентами. Администрации необходимо хранить и обрабатывать информацию о пользователях, включая контактные данные, историю заказов и предпочтения по способам оплаты. Данные используются для создания заказов, формирования отчётности и обеспечения обратной связи с клиентами. При большом количестве пользователей ручная обработка информации становится неудобной и требует значительных временных затрат.

Одним из наиболее важных процессов является управление заказами. Каждый заказ представляет собой сделку по обмену криптовалюты на фиатные средства по определённому курсу. Для каждой сделки существует ограничение по минимальной и максимальной сумме, которое должно автоматически контролироваться системой. Кроме того, для каждого заказа должен быть указан статус, отражающий текущий этап обработки: ожидание оплаты, подтверждение транзакции, завершение или отмена.

В разрабатываемой системе предусмотрена возможность общения клиентов с поддержкой через встроенный чат. Наличие подобного функционала позволяет пользователям задавать вопросы, уточнять детали сделки и получать оперативную помощь непосредственно внутри платформы. Это повышает удобство использования системы и обеспечивает дополнительное взаимодействие между клиентами и администрацией сервиса.

Важной частью деятельности сервиса являются финансовые операции. Клиенты отправляют криптовалюту на указанный адрес, после чего система проверяет поступление средств и инициирует выплату фиатных средств. Для подтверждения оплаты необходим мониторинг блокчейн-транзакций и автоматическая проверка статусов платежей. При ручной обработке повышается вероятность ошибок и увеличивается время обслуживания клиентов. В связи с этим система должна обеспечивать автоматическую проверку транзакций и обновление статусов заказов.

Дополнительно в сервисе осуществляется учёт платёжных реквизитов и криптовалютных адресов. При поступлении новых средств необходимо регистрировать транзакцию, фиксировать ответственного сотрудника и обновлять баланс. Данный процесс особенно важен для обеспечения прозрачности финансового учёта и возможности отслеживания движения средств.

Для реализации перечисленных процессов в системе предусмотрено разграничение пользователей по ролям. Каждая роль определяет набор доступных функций и уровень доступа к данным.

Клиент является основной категорией пользователей системы. Пользователь данной роли может просматривать курсы обмена, создавать заказы, оплачивать их через криптокошелёк и общаться со службой поддержки.

Администратор обладает полным доступом ко всем функциям системы. Он отвечает за управление пользователями, настройку курсов, контроль заказов, управление базой данных и просмотр всей отчётности.

В ходе анализа предметной области было установлено, что деятельность сервиса обмена криптовалют включает большое количество взаимосвязанных процессов, требующих централизованного управления и автоматизации. Использование современной системы позволит существенно повысить скорость обработки информации, снизить вероятность ошибок, улучшить качество обслуживания клиентов и обеспечить эффективное взаимодействие между пользователями и администрацией сервиса.

**1.2 Анализ существующих решений**

В процессе выполнения дипломного проекта был проведён анализ существующих программных решений, используемых для обмена криптовалют, управления цифровыми активами и автоматизации деятельности обменных сервисов.

Одной из наиболее популярных платформ является Binance. Данный сервис представляет собой крупнейшую международную криптовалютную биржу и предоставляет пользователям возможность обмена цифровых активов, торговли и хранения средств. Несмотря на развитый функционал, система ориентирована преимущественно на биржевую торговлю и не предоставляет специализированных средств для прямого обмена криптовалют на фиатные средства с выводом на банковские карты.

Также был рассмотрен сервис BestChange, который представляет собой мониторинг обменных пунктов. Платформа позволяет сравнивать курсы различных обменников и выбирать наиболее выгодное предложение. Однако BestChange является агрегатором и не осуществляет непосредственное проведение обменных операций, а также не предоставляет инструментов для автоматизации внутренних процессов конкретного сервиса.

В качестве аналога был проанализирован сервис ChangeNOW. Данная система обеспечивает мгновенный обмен криптовалют без необходимости регистрации. Платформа поддерживает большое количество криптовалют и предоставляет фиксированный курс на момент создания заявки. Вместе с тем функционал ориентирован преимущественно на обмен криптовалют между собой и не включает полноценной поддержки вывода фиатных средств, персонализированного обслуживания и встроенной системы коммуникации с клиентами.

Дополнительно был рассмотрен сервис LocalCoinSwap, используемый для организации Peer-to-Peer обмена криптовалют. Система позволяет создавать объявления о покупке и продаже цифровых активов, общаться с контрагентами и проводить сделки. Недостатком решения является отсутствие централизованного контроля над курсами и ликвидностью, а также необходимость самостоятельного поиска контрагентов.

Также был выполнен анализ открытых проектов, размещённых на платформе GitHub, предназначенных для создания криптовалютных обменников. Большинство таких решений обладают ограниченным функционалом, требуют существенной доработки и не обеспечивают комплексной автоматизации всех бизнес-процессов сервиса.

По результатам проведённого анализа были выявлены следующие недостатки существующих решений:
— отсутствие единой системы управления деятельностью обменного сервиса;
— ограниченные возможности интеграции с криптокошельками;
— отсутствие встроенной системы проверки транзакций;
— отсутствие персонализированной поддержки клиентов через чат;
— отсутствие механизмов автоматической генерации отчётности;
— необходимость использования нескольких отдельных сервисов для решения различных задач;
— высокий порог входа для начинающих пользователей.

Кроме того, большинство рассмотренных программных продуктов решают лишь отдельные задачи: предоставление курсов, проведение обмена или хранение активов. Комплексного решения, объединяющего создание заказов, оплату через криптокошелёк, проверку транзакций, поддержку клиентов и формирование отчётности, выявлено не было.

На основании проведённого анализа было принято решение о разработке собственной автоматизированной системы обмена криптовалют. Разрабатываемое решение позволит объединить весь необходимый функционал в единой информационной среде, что повысит эффективность работы сервиса, сократит затраты времени на выполнение повседневных операций и обеспечит удобное взаимодействие между администрацией и клиентами.

**1.3 Формирование основных требований к разрабатываемому программному продукту**

На основании анализа предметной области и существующих решений были сформированы основные требования к разрабатываемой автоматизированной системе обмена криптовалют.

Главной целью разрабатываемого программного продукта является автоматизация процессов создания и обработки заказов, проведения платежей, проверки транзакций и формирования отчётной документации.

Система должна обеспечивать регистрацию и авторизацию пользователей. Для повышения уровня безопасности доступ к административной панели должен осуществляться по токену, генерируемому на основе секретного ключа. После успешной авторизации пользователь получает доступ только к тем функциям системы, которые соответствуют его роли.

Разрабатываемое приложение должно поддерживать ролевую модель доступа. Для каждой категории пользователей необходимо определить набор доступных возможностей и ограничений. Реализация подобного механизма позволит обеспечить защиту данных и предотвратить выполнение несанкционированных действий.

Система должна предоставлять функционал для создания и обработки заказов на обмен криптовалют. При создании заказа система должна автоматически рассчитывать сумму к получению на основе текущего курса и комиссии сервиса. Дополнительно необходимо реализовать механизм автоматического обновления курсов в реальном времени.

Для обеспечения возможности оплаты через криптокошелёк необходимо реализовать интеграцию с плагином-расширением Phantom для блокчейна Solana. Пользователь должен иметь возможность подключить свой кошелёк и отправить средства непосредственно через интерфейс системы.

Система должна автоматически проверять статус блокчейн-транзакций. После отправки средств клиентом система должна выполнять проверку подтверждения транзакции в сети и обновлять статус заказа. Для обеспечения надёжности необходимо реализовать поддержку нескольких провайдеров блокчейн-данных.

Для повышения удобства взаимодействия пользователей с администрацией необходимо реализовать встроенный чат поддержки. Доступ к чату осуществляется через виджет на странице сервиса. Пользователи должны иметь возможность отправлять сообщения и получать ответы от службы поддержки в режиме реального времени.

В системе должен присутствовать модуль отчётности, обеспечивающий просмотр истории заказов с возможностью фильтрации по статусам, датам и суммам. Администратор должен иметь доступ к полной информации о всех заказах сервиса.

В качестве платформы разработки выбрана связка FastAPI (Python) для серверной части и JavaScript для клиентской. Данная технология обеспечивает высокую производительность, асинхронную обработку запросов и возможность реализации современной веб-архитектуры. Для взаимодействия с базой данных используется SQLAlchemy совместно с СУБД SQLite.

Разрабатываемая система должна обладать удобным пользовательским интерфейсом. Интерфейс должен обеспечивать простую навигацию между разделами, поддержку адаптивной структуры страниц и корректное отображение данных.

Программный продукт должен поддерживать многопользовательский режим работы и обеспечивать стабильное функционирование при одновременной работе нескольких пользователей.

Таким образом, сформированные требования определяют структуру и функциональные возможности будущей автоматизированной системы и обеспечивают соответствие разрабатываемого программного продукта задачам сервиса обмена криптовалют.
